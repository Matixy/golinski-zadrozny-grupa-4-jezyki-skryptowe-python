Lista zadań 1.1 "Start pracy zespołowej: Python + GitHub"
Skład zespołu: Paweł Goliński, Mateusz Zadrozny

Cel projektu: Celem niniejszego projektu jest praktyczne zapoznanie się z podstawami współpracy zespołowej w środowisku Python przy użyciu systemu kontroli wersji GitHub. 

Zadanie obejmuje: 
- Przygotowanie środowiska pracy i repozytorium.
- Przećwiczenie pracy na branchach, commitach oraz procesie Pull Request i Code Review.
- Stworzenie prostego programu w Pythonie realizującego podstawowe operacje i funkcje.
- Implementację testów jednostkowych dla stworzonego kodu.

Funkcjonalności programu:
Aplikacja jest prostym skryptem w Pythonie, który: 
- Wyświetla nazwę zespołu oraz listę jego członków.
- Potwierdza poprawne uruchomienie projektu.
Posiada dodatkowe funkcje, takie jak:
- Zliczanie liczby członków zespołu.
- Formatuje powitania dla użytkowników.
- Prezentuje dane w uporządkowanej, czytelnej formie.

Struktura projektu został zorganizowany zgodnie z poniższą strukturą:
src/ – kod źródłowy programu
.tests/ – testy sprawdzające poprawność działania funkcji
.docs/ – dokumentacja techniczna i notatki projektowe
.data/ – pliki z danymi wejściowymi
.requirements.txt – lista bibliotek niezbędnych do uruchomienia projektu.

Instrukcja uruchomienia 
Wymagania: Zainstalowany Python (zalecana wersja 3.x).
Kroki instalacji:
1. Sklonuj repozytorium na swój komputer. 
2. Utwórz lokalne środowisko wirtualne: python -m venv venv
3. Aktywuj środowisko wirtualne: venv\Scripts\activate (WINDOWS) source venv/bin/activate (macOS/Linux)
4. Zainstaluj wymagane biblioteki (jeśli są obecne):pip install -r requirements.txt
Uruchomienie programu:
Aby uruchomić główną aplikację, wykonaj polecenie: python src/main.py
